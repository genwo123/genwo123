##  지원동기

저는 2년간 언리얼 엔진으로 게임 클라이언트를 개발하며 UI 시스템과 팀 협업을 경험해왔습니다.

**미신전(2024.09~10)**은 숙명여대 졸업전시회를 2개월 안에 완성해야 하는 프로젝트였습니다. 2명의 개발자가 상호작용 시스템과 인벤토리를 빠르게 구현하는 데 집중했고, 단기 개발을 위해 Common UI 플러그인을 선택했습니다. 짧은 기간이라 아쉬운 부분도 있었지만, 빠른 개발과 데드라인 관리를 경험할 수 있었습니다.

**하모니아(2025.02~10)**는 6명 팀(기획1, 아티스트3, 사운드1, 개발자1)의 유일한 프로그래머로 10개월간 진행한 퍼즐 게임입니다. 레이저 퍼즐 시스템, 퀘스트 UI, 대화 시스템, 노트 시스템을 전담하며 DataTable 기반 재사용 컴포넌트를 설계했습니다. 비개발자 5명과 협업하며 Git 가이드를 작성하고, 기술적 제약을 쉽게 설명하는 법을 익혔습니다. 숙명여대 졸업전시회를 성공적으로 마쳤고, 현재 여러 공모전에 출품 중입니다.

**브릿지런(2024.11~)**은 멀티플레이어 게임으로 기획했지만, 네트워크 프로그래밍을 제대로 모른 채 시작했다가 벽에 부딪혔습니다. "이렇게 야매로 하면 안 되겠다"는 생각에 지금은 RPC, Replication, 서버-클라이언트 아키텍처를 제대로 공부하며 천천히 진행 중입니다. SOLID 원칙을 적용한 리팩토링 과정에서 "코드가 짧다고 좋은 게 아니라, 독립적이고 연쇄 작용 없이 사용할 수 있는 게 더 효율적"이라는 걸 깨달았습니다.

**게임 외 협업 경험도 다양합니다.** AWS 리빙랩 해커톤에서는 7명 팀의 아키텍처 설계와 프론트엔드를 맡아 대상을 받았고, UFO's 프로젝트에서는 9명 팀의 PM으로 QR 주문시스템을 개발해 은상을 받았습니다. 3년 연속 동아리장을 하며 50명 회원을 관리하고 코딩 스터디를 운영했습니다. 여러 프로젝트에서 리더 역할을 맡으며 적극적으로 의견을 내고, 막히는 부분이 있어도 어떻게든 해결책을 찾아내는 자세를 길렀습니다.

**그런데 개발을 계속하다 보니 생각이 바뀌기 시작했습니다.**

언리얼 엔진이다 보니 Blueprint와 C++을 혼용하며, 성능이 중요한 부분은 C++로, 직관적인 작업은 Blueprint로 처리하는 방식을 익혔습니다. C++로 메모리 관리와 성능 최적화를 고민할 때마다, "이 코드가 실제로 어떤 하드웨어에서 돌아가는지" 궁금해졌습니다. 게임 엔진은 GPU, CPU를 잘 추상화해주지만, 오히려 그래서 실제 기계와의 거리가 멀게 느껴졌습니다.

그러다 자동차 전장 시스템, 특히 **임베디드 시스템**을 접하게 되었습니다. 센서와 직접 통신하고, 제한된 메모리와 CPU 자원 안에서 실시간으로 작동하는 시스템. 게임 엔진처럼 추상화된 환경이 아니라, 하드웨어와 맞닿아 있는 로우레벨 개발. 이쪽에 점점 관심이 생겼습니다.

**솔직히 말하면, 제가 해온 게임 프로젝트가 임베디드 소프트웨어 개발에 직접적으로 얼마나 도움이 될지는 잘 모르겠습니다.** 하지만 차량 인포테인먼트 시스템이나 임베디드 HMI 개발 분야를 보면서, 현재 제가 가진 능력을 살릴 수 있는 곳이 여기라는 확신이 들었습니다.

먼저, **C++ 경험**입니다. 언리얼 엔진에서 C++를 사용하며 포인터 관리, 메모리 할당/해제, 캐스팅, 컴포넌트 생명주기 관리 등을 다뤄봤습니다. 물론 게임 개발 중심이었지만, C++ 자체에 대한 이해도가 있어 임베디드 환경의 C++에 빠르게 적응할 수 있을 것이라고 생각합니다.

다음으로 **UI 시스템 설계 경험**입니다. 하모니아에서 퍼즐 UI, 퀘스트 시스템, 노트 시스템을 DataTable 기반으로 설계하며 재사용 가능한 컴포넌트를 만들었습니다. 차량의 클러스터 계기판이나 HMI 시스템도 결국 사용자에게 정보를 직관적으로 전달하는 UI이기 때문에, 이 경험이 도움이 될 것이라 생각합니다.

무엇보다 **협업과 문제 해결 능력**이 제 강점입니다. 10개월간 비개발자들과 협업한 하모니아, 대상을 받은 AWS 해커톤, PM을 맡은 UFO's 프로젝트까지 다양한 팀에서 일하며 적극적으로 의견을 내고 막히는 부분을 해결하는 법을 익혔습니다.

제가 만든 시스템이 실제 제품에 탑재되어 현실에서 작동한다는 것. 가상의 게임이 아닌, 실제 하드웨어와 만나 사람들의 일상에 도움이 되는 소프트웨어. 그게 지금 제가 하고 싶은 일입니다. 임베디드 시스템은 처음이지만, 하드웨어와 가까운 곳에서 코드를 작성하고 싶다는 마음이 확실합니다.

##  입사 후 포부

입사 초반 6개월은 완전히 새로운 환경에 적응하는 데 집중하겠습니다. 

먼저 **임베디드 시스템 학습**입니다. 실시간 통신 프로토콜, 센서 데이터 처리, 하드웨어와의 인터페이스 구조를 공부하고, 해당 분야의 표준과 규격도 익히며 기본기를 쌓겠습니다.

다음으로 **하드웨어 제약 속 개발에 적응**하겠습니다. 게임 엔진의 추상화된 환경에서 벗어나, 실제 하드웨어 리소스 제약 속에서 최적화하는 방법을 배우겠습니다. RTOS 환경에서의 태스크 관리, 메모리 할당 전략을 익히며, 기존 코드베이스를 별도 브랜치로 분석하고 선배님들께 적극적으로 질문하겠습니다.

**C++ 실력을 임베디드 수준으로 끌어올리는 것**도 중요한 목표입니다. 언리얼에서 사용한 C++는 게임 엔진 위에서 돌아가는 것이었지만, 여기서는 하드웨어에 훨씬 가까운 C++를 다루게 될 것입니다. 포인터, 메모리 관리, 구조체 같은 기본기는 있으니, 임베디드 특유의 하드웨어 제어나 인터럽트 처리를 빠르게 습득하겠습니다.

제가 가진 강점도 활용하겠습니다. 하모니아에서 10개월간 6명 팀과 협업하며 쌓은 경험, Git 형상관리 가이드 작성, 비개발자 소통 능력은 하드웨어 팀과의 커뮤니케이션에 도움이 될 것입니다. 여러 프로젝트에서 막히는 부분을 해결하며 익힌 문제 해결 능력도 새로운 환경에서 빛을 발할 것이라 믿습니다.

솔직히 게임 개발과 임베디드 개발은 다른 분야입니다. 하지만 "하드웨어와 가까운 곳에서 코드를 작성하고 싶다"는 제 방향성은 확실합니다. 선배님들께 적극적으로 배우며, 임베디드 환경에서의 C++ 개발 역량을 빠르게 끌어올리겠습니다.

장기적으로는 임베디드 시스템 분야에서 사용자 경험과 하드웨어 효율성을 모두 고려하는 개발자로 성장하고 싶습니다. 사용자가 직관적으로 이해하고 안전하게 사용할 수 있는 시스템을 만드는 것. 그것이 제 목표입니다.
